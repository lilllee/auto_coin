### [ ] B2. 검증 인프라 portfolio-aware 확장 (CSMOM 구현 전 의무 선행)

**CSMOM은 strategy 추가가 아니라 portfolio engine 확장이다. 검증 인프라 없이는 CSMOM도 다시 "숫자 좋아 보이는 in-sample"에 머무름.**

- [ ] `backtest/portfolio_runner.py` 신규: universe(티커 dict) 입력 · rank 계산 · weekly rebalance · portfolio equity 출력
- [ ] `backtest/walk_forward.py` → universe-level WF 모드 추가 (포트폴리오 기준 avg excess / positive window / trade count 집계)
- [ ] `web/models.py::DailySnapshot` → `portfolio_equity_krw`, `portfolio_excess_vs_bnh` 컬럼 추가
- [ ] KPI 페이지에 포트폴리오 기준 누적 수익률 메인 지표로 표기
- [ ] `ExecutionContext` (또는 동등 구조체)에 volatility-scaled sizing 파라미터 추가 (`risk_budget_krw`, `atr_window_for_sizing`)

**근거**: 후보 A/D는 cross-sectional 랭킹이 필요, B/C는 vol sizing이 필요. 현 인프라로는 둘 다 검증 불가.
