## regime-only baseline 설계

두 가지 순수 regime-only 전략. cross-sectional ranking 없음:

| 이름 | 동작 |
|---|---|
| `regime_equal_weight` | regime ON → universe 전 ticker 동등 가중 (risk_budget%), regime OFF → cash |
| `regime_btc_only` | regime ON → BTC 100% (risk_budget), regime OFF → cash |

두 baseline 모두 CSMOM 의 `_is_risk_on` 헬퍼를 재사용 (regime 로직 공유), WF `SignalFactory` 와 호환.

### 결정적 관찰

- Regime-only baseline 도 전체 avg_excess 는 양수 근처 (+0.59% / -0.62%) — 이는 "regime OFF 기간 B&H 의 손실을 피해서" 생긴 flat 효과.
- **단 strategic-only 로 보면 regime-only baseline 은 -5% ~ -7% 손해** — regime ON 상태에서 단순 동등가중/단일자산은 실제로는 B&H 에게 **진다**.
- CSMOM v1 strategic 은 +1.01% → baseline 대비 incremental **+7.80%p**.

→ **CSMOM 의 momentum selection 은 baseline 대비 진짜 alpha 를 추가한다**.

### Decomposition

- v1 전체 avg_excess +4.03% 중 "regime-flat 윈도우 기여" 는 약 **+3.02%p** (overall - strategic)
- v1 strategic 대 regime_equal baseline strategic: **incremental selection alpha +7.80%p**
- v2 strategic 대 regime_equal baseline strategic: incremental **+5.51%p**

→ **v2 도 baseline 대비 incremental alpha 를 유지**, 다만 v1 대비 2.3%p 감소.

## Selection alpha 존재 여부

**핵심 답변**:

1. **v1 의 성과 대부분이 regime-only 로 설명되는가?** → **아니다**. v1 의 overall excess 중 flat 기여는 3.02%p, 하지만 strategic 기여도 +1.01% (baseline -6.79% 대비 +7.8%p).
2. **v2 는 v1 보다 수익 낮아져도 stability 좋아지는가?** → **Yes**. avg_excess +4.03% → +2.51%, stability 27.8% → 50.0%. strategic 은 +1.01% → -1.28% 로 악화 (edge 희생).
3. **v2 가 regime-only 보다 의미 있는 selection alpha 를 추가하는가?** → **Yes**. incremental +5.51%p (strategic 기준).
4. **CSMOM 은 사실상 "regime overlay" 에 불과한가?** → **아니다**. Selection alpha 존재가 데이터로 입증됨. 단 현 구현은 stability-edge trade-off 에 걸려 있음.

## RCDB 병렬 착수 여부

사용자 설정 "판단 기준" 매칭:

| 기준 | v2 결과 |
|---|---|
| CSMOM v2 도 stability hard-fail | **❌ 아니다** — stability 50% 로 hard-fail 탈출 (soft-warn 구간) |
| selection alpha 가 regime-only 대비 미미 | **❌ 아니다** — incremental +5.5~7.8%p |
| avg_excess 대부분이 flat-in-bear 에서 | **△ 부분적** — v1 3.02%p 가 flat, 단 strategic +1% 도 존재 |
| v2 가 pass target 못 미침 | **△ 부분적** — avg_excess +2.51% (pass), positive 61% (pass), trades 70 (pass), stability 50% (soft-warn), strategic -1.28% (실패) |

### 결정: **RCDB 병렬 착수 권장하지 않음 · v3 시도 후 결정**

근거:
- Selection alpha 존재가 확인되어 CSMOM 을 "단순 regime overlay" 로 단정할 수 없음
- v2 에서 stability 가 실제로 개선되는 메커니즘 확인 (grid 축소 → stability 상승)
- 하지만 v2 의 strategic avg 음수 전환은 lookback=30 제거의 반작용
- **다음 시도**: v3 로 lookback ∈ {45, 60}, top_k ∈ {2}, rebal ∈ {14}, regime_ma ∈ {100} 등 **더 좁은 grid** 로 stability pass target (≥60%) 돌파 + strategic edge 유지 가능한지 확인

RCDB 병렬 착수는 v3 에서도 stability / strategic 둘 중 하나 해결 안 될 경우 (HOLD → REVISE 반복) 로 미룸.

### **HOLD — v3 추가 실험 후 결정**

- 구조는 의미 있지만 파라미터 공간이 여전히 넓어 stability hard-fail 임계 근처
- 한 번 더 grid 축소 (v3) 로 stability pass + strategic 양수 동시 달성 시도 가치 있음
