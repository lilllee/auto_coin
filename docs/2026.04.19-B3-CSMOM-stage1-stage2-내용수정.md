## CSMOM v1 signal 설계

### 파라미터 (CsmomParams)

| 필드 | 기본값 | 역할 |
|---|---|---|
| `lookback_days` | 60 | 과거 수익률 계산 기간 |
| `top_k` | 3 | 보유 상위 K 종목 |
| `regime_enabled` | True | 레짐 필터 활성 |
| `regime_ticker` | "KRW-BTC" | 레짐 기준 자산 |
| `regime_ma_window` | 100 | SMA 기간 (daily) — "BTC close > SMA(N)" 이면 risk-on |

**rebal_days · hold_N · risk_budget** 은 `PortfolioContext` 에서 주입 (이중 정의 방지).

### 결정 로직 (순수 함수)

```
1. _is_risk_on(candles, params) → False 이면 → 전원 flat ({})
2. 각 ticker 의 (current / close_{t-lookback}) - 1 → momentum_score
3. score > 0 종목만 후보 (no-shorting)
4. rank 상위 min(top_k, ctx.hold_N) 선택
5. 각 선정 종목에 risk_budget / K 동등 가중
```

**의도적 단순화**:
- 레짐: close > SMA(N) 단일 조건 (slope 없음)
- sizing: equal-weight × risk_budget (ATR 가중 없음)
- catastrophic stop / 개별 exit 없음 → rebalance 로만 전환
- survivorship bias 보정 없음 → portfolio_runner 의 교집합 의존

### Factory 호환

`csmom_factory(params_dict) → (signal, context_overrides)` 구조.
portfolio_walk_forward.SignalFactory 와 호환 → Stage 4 에서 바로 재사용.

## Stage 2 · in-sample 2y backtest

### 실행 조건
- Universe: KRW-BTC, KRW-ETH, KRW-XRP, KRW-SOL, KRW-DOGE (5개)
- 기간: 2024-04-19 → 2026-04-18 (730 일봉)
- fee 0.05% · slippage 0.05% · initial 1,000,000 KRW · risk_budget 0.8
- Default params: lookback=60, top_k=3, rebal_days=7, regime_ma=100

### Default 결과

| 지표 | 값 | PLAN §4 Stage 2 기준 | 판정 |
|---|---:|---|---|
| Cumulative Return | **+110.64%** | — | — |
| B&H (equal-weight) | +22.63% | — | — |
| **Excess vs B&H** | **+88.01%** | ≥ +10% | ✅ pass target |
| MDD | -26.73% | ≥ -30% | ✅ pass target |
| Sharpe (daily→ann) | **1.31** | ≥ 0.8 | ✅ pass target |
| Trade count | 152 | ≥ 60 | ✅ pass target |
| Rebalances | 105 | — | — |
| Avg trades / rebal | 1.45 | — | 적절한 회전 |

**4/4 Pass target 모두 충족 · Hard-fail 전무 · Soft-warning 전무**.

### Sanity 체크 (인접 파라미터 · regime 효과)

| Variant | Excess | MDD | Sharpe | Trades | 결과 |
|---|---:|---:|---:|---:|---|
| **regime OFF** | +53.54% | -31.78% | 0.84 | 192 | Soft-warn (MDD) — regime 가치 입증 |
| lookback=30 | +223.94% | -17.34% | 1.97 | 145 | 모두 pass (오히려 더 강함) |
| lookback=90 | +117.21% | -26.27% | 1.47 | 141 | 모두 pass |
| top_k=2 | +135.91% | -23.92% | 1.47 | 112 | 모두 pass |
| rebal=14 | +58.87% | -27.70% | 1.08 | 84 | 모두 pass |

**관찰**:
1. **Regime filter 가 실질적으로 작동**: OFF 시 Excess 88% → 53%, MDD -26.73% → -31.78%.
   "BTC 하락장 노출을 잘라내 수익과 MDD 둘 다 개선" 이라는 PLAN 의 가설을 1차 지지.
2. **인접 파라미터 전부 hard-fail 피함** → 한 점 sweet spot 의존 아님.
3. **lookback=30 이 default 보다 강함** — Stage 3 grid sweep 에서 더 탐색할 가치.
