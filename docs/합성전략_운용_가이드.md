# SMA200 EMA+ADX 합성 전략 운용 가이드

## 개요

이 문서는 `sma200_ema_adx_composite` 전략의 **설정, 백테스트, 실전 운영**을 위한 완전한 가이드입니다.

**전략 핵심:**
- **2단계 판단**: SMA200 레짐 필터(위험 온/오프) + EMA+ADX 추세 진입
- **모두 일봉 기반**: 일단위 확정봉(shift=1) 사용
- **자동 청산**: price < SMA200이면 즉시 SELL (레짐 청산)
- **손절/한도**: 외부 RiskManager가 처리 (-2%, -3%, kill-switch)

> **⚠️ 백테스트 한계:** 현재 백테스트는 합성 전략의 진입/청산 신호 검증 중심이며, RiskManager 기반 손절(-2%)/일일한도(-3%)/쿨다운(30분)은 반영되지 않습니다. 백테스트 결과는 실전 PnL 추정치가 아닌 전략 신호 품질 검증용으로만 활용하세요.

## 1. 추천 기본 설정값

### 1-1. 전략 파라미터

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| **sma_window** | 200 | 보고서 기준 표준 risk-on/off 필터. 기본값 유지 권장 (초기 검증 완료 전 임의 변경 비권장). |
| **ema_fast_window** | 27 | 보고서 추천 단기 EMA (골든크로스 판정용). 기본값 유지 권장. |
| **ema_slow_window** | 125 | 보고서 추천 장기 EMA (골든크로스 판정용). 기본값 유지 권장. |
| **adx_window** | 90 | 보고서 추천 ADX 기간 (추세 강도 판정). 기본값 유지 권장. |
| **adx_threshold** | 14.0 | 추세 강도 임계값. 스윕 테스트: 10~20 권장 |

**파라미터 설정 원칙:**
- 초기에는 **기본값 그대로** 사용
- 백테스트로 검증 후 **adx_threshold만 미세조정** (10, 12, 14, 16, 18, 20)
- SMA/EMA/ADX 윈도우는 보고서 기준이므로 **초기 검증 완료 전 임의 변경 비권장** (검증 없이 변경하면 보고서 기반 논거가 무효화됨)

### 1-2. 추천 종목

| 단계 | 종목 | 이유 |
|------|------|------|
| **1차 검증** | KRW-BTC 단일 | 가장 유동성 높고 데이터 풍부, SMA200 필터와 궁합 최고 |
| **2차 확장** | KRW-BTC + KRW-ETH | 2종목 포트폴리오. ETH도 추세 사이클이 명확 |
| **3차 확장** | KRW-BTC, KRW-ETH, KRW-SOL | 3종목 최대 동시 보유. 알트는 노이즈 많음 |

**주의:**
- 알트코인(SOL, LUNA 등)은 SMA200 필터 효과가 약할 수 있음
- 초기 검증은 반드시 **BTC 단일**로 시작할 것

### 1-3. 필수 캔들 수 (count)

합성 전략이 유효하려면 충분한 역사 데이터가 필요합니다:

- **SMA200**: 최소 200봉 (+ 50 여유) = 250봉
- **EMA125**: 최소 125봉 (+ 워밍업)
- **ADX90**: 최소 90봉 (+ 워밍업)

**권장: count = 300봉**

봇에서 자동 계산:
```python
required_count = max(sma_window=200, ema_slow_window=125, adx_window=90) + 50
# = 250 최소, 보수적으로 300 권장
```

**실전 의미:**
- 웹 UI/CLI에서 `--count 300` 또는 자동값 사용
- API 응답 시간: pyupbit get_ohlcv(count=300) ≈ 0.5초
- 초기 부팅 시에만 300봉 fetch, 이후 매 tick마다 +1 업데이트

### 1-5. 손절/청산 전략

합성 전략의 손절과 청산은 여러 계층으로 구성됩니다:

#### 1단계: SMA200 레짐 청산 (가장 빠름)

```
price < SMA200 → 즉시 SELL (보유 포지션도 청산)
```

- **시점**: 매 tick마다 체크 (60초 주기)
- **특징**: 추세 반전을 감지한 조기 청산
- **예시**: BTC가 하락장에 진입하면 "위험 신호" → 강제 청산

#### 2단계: 개별 손절 (-2%)

```
(current_price - avg_entry_price) / avg_entry_price <= -2%
```

- **주체**: RiskManager에서 판정 (strategy 신호와 무관하게 독립 실행)
- **우선순위**: BUY 신호보다 높음 (손절이 먼저)
- **진입가**: 각 종목별 평균 진입가

#### 3단계: 일일 손실 한도 (-3%)

```
포트폴리오 일일 PnL 합계 <= -3%
```

- **적용**: 신규 BUY는 차단, 기존 SELL은 실행
- **리셋**: 09:00 KST (일일 초기화)
- **멀티 종목**: 모든 종목의 손익을 합산하여 판정

#### 4단계: 시간 청산 (08:55 KST) — 권장: 비활성화

```
08:55 KST에 모든 포지션 강제 청산
```

**중요 설계 결정:**
- 변동성 돌파는 일중 전략이라 08:55 청산이 적절
- **합성 전략은 추세추종**이므로 다음날 자동 청산은 **비적합**
- **권장**: exit_hour_kst를 **비활성화** (또는 -1로 설정) → 포지션을 추세가 끝날 때까지 유지

**현재 제약:**
- exit_hour_kst는 전역 설정 (전략별 분리 불가)
- 프리셋 B/C에서는 명시적으로 OFF 설정

#### 5단계: Kill-Switch (수동 비상 차단)

```
Kill-switch = ON → 신규 BUY 차단, 기존 청산은 허용
```

- **용도**: 급변하는 시장 상황에서 수동 개입
- **웹 UI**: /settings/risk → "Kill-switch" 토글
- **cli**: `--live --kill-switch` (사용 권장 안 함)

#### 6단계: 쿨다운 (30분)

```
청산 후 30분간 신규 진입 차단
```

- **목적**: 청산 후 즉시 재진입 방지 (추세 반전 확인용)
- **일봉 전략에서의 의미**: 30분은 같은 봉 내이므로 실질적 영향 낮음
- **향후 1H 전략으로 전환 시** 중요해짐

### 4-1. 페이퍼 트레이딩 검증 (최소 1개월)

| 항목 | 확인 사항 | 체크 |

| **쿨다운 적용** | 청산 후 30분간 신규 진입 없음 | ☐ |
| **데이터 정합성** | 캔들 가격 vs 현재가 오차 < 1% | ☐ |

### 4-2. 시스템 리스크 점검

| 리스크 | 대응 |
|--------|------|
| **API 속도 저하** | API 응답 시간 로그 확인. 1초 이상이면 throttle 조정 |

## 10. 자주 묻는 질문 (FAQ)

### Q1. 프리셋 A와 B 중 어느 것으로 시작해야 하나?

**답:** 무조건 **프리셋 A**부터 시작하세요.
- 전략이 정말 동작하는지 기본 검증
- 리스크 최소 (10% 투입, -1.5% 손절)
- 1-2주면 충분한 신뢰도 확보 가능

### Q2. 시간 청산(08:55)을 ON 해야 하나?

**답:**
- **프리셋 A**: **OFF 기본값**. 합성 전략은 추세추종이라 시간 청산이 비적합. VB 대비 비교 실험 시에만 ON으로 설정 가능.
- **프리셋 B/C**: **OFF 권장** (합성 전략은 추세 유지가 핵심)

현재 exit_hour_kst는 전역 설정이라 두 전략이 충돌할 수 있습니다.
향후 전략별 설정 분리 계획.

### Q3. ADX threshold를 왜 14.0으로?

**답:** 보고서 기준 추천값입니다.
- 14 미만: 추세 약한 구간도 진입 → 승률 낮지만 거래 많음
- 14~16: 균형 (default)
- 16 이상: 추세 강한 구간만 진입 → 승률 높지만 거래 적음

백테스트로 최적값 찾는 것 권장.

### Q4. 손절(-2%)과 일일한도(-3%) 차이가 뭔가?

**답:**
- **손절(-2%)**: 개별 종목당 진입가 대비 손실
- **일일한도(-3%)**: 포트폴리오 전체 일일 손실

예시:
```
09:00 KRW-BTC 진입 @ 50,000K (50K 투입)
       현재 손실: -1% (개별 손절 안 함)
11:00 KRW-ETH 진입 @ 2,000K (50K 투입)
       현재 손실: -2% (개별 손절 안 함)
       포트폴리오 손실: -1.5% (일일한도 안 함)
13:00 BTC 추가 하락: -2.1% (개별 손절 발동)
       → BTC 즉시 청산
```

### Q5. 멀티 종목(BTC+ETH)일 때 손절이 한 개만 발동되면?

**답:** 정상입니다.
- BTC -2% 손절 발동 → BTC만 청산
- ETH 보유 중이고 손실 없으면 → ETH 유지
- 각 종목이 독립적으로 손절 판정

### Q6. 방법 B(1H)는 언제 구현되나?

**답:**
- 현재 일봉 전략 안정화 우선
- Q3 2026 예정 (구체적 일정 TBD)
- 1H 데이터 처리, 멀티타임프레임 로직, 백테스트 지원 필요
- 별도 가이드 문서 제공 예정

### Q7. 페이퍼에서 Red이면 실거래도 Red일 가능성 높나?

**답:** 예, 매우 높습니다.
- 페이퍼와 실거래는 같은 로직
- 페이퍼 손실 = 설정/전략 문제일 가능성 높음
- 반드시 페이퍼에서 Green 확보 후 실거래 전환

### Q8. 이미 실거래(변동성 돌파) 중인데 합성으로 바꿔도 되나?

**답:**
- **기존 포지션이 없으면**: 바로 전환 가능
- **기존 포지션이 있으면**: 먼저 청산 후 전환

주의: 전략 변경 전 기존 strategy.json 백업 권장.

### Q9. 합성 전략과 변동성 돌파를 동시에 실행할 수 있나?

**답:** **절대 금지.**
- 봇은 단일 전략만 지원
- 포지션 중복 진입, 손절 충돌 등 문제 발생
- 한 번에 하나의 전략만 사용

## 13. 요약: 핵심 3가지

### 1️⃣ SMA200 필터 (위험 온/오프)

```
price >= SMA200 → 위험 ON (진입 가능)
price < SMA200  → 위험 OFF (청산 강제)
```

이 필터가 하락장을 자동으로 피해주는 마법의 메커니즘입니다.

### 2️⃣ EMA+ADX 진입 (추세 진입)

```
ema27 > ema125 AND adx >= 14.0 → BUY
```

골든크로스는 진입, ADX는 추세 강도 확인. 둘 다 만족해야 진입.

### 3️⃣ 다층 손절 (안전장치)

```
개별 손절(-2%) > 일일한도(-3%) > 시간청산(off) > kill-switch
```

여러 단계의 안전장치가 손실을 제한합니다.
