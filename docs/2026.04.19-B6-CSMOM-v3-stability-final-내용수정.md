## strategic-only 평가 방식

B6 부터 전체 avg_excess 가 아니라 **strategic-only** 지표를 메인으로:

- `strategic_avg_excess` = avg(excess) across windows with test_trades > 0
- `strategic_positive_ratio` = fraction with excess > 0 among trading windows
- `incremental_selection_alpha` = strategic_avg(model) − strategic_avg(regime_equal baseline)
- `flat_avg` = avg(excess) across windows with test_trades = 0 (contribution 분리)

전체 avg_excess 에 "flat-in-bear" 효과가 섞이는 것을 배제.

## stability 변화 분석 — **예상과 반대**

**가설**: grid 축소 → winning params 선택지 감소 → 같은 조합이 자주 선택 → stability 상승.

**실제 관찰**:
- v1 (36) → stability **27.8%**
- v2 (12) → stability **50.0%** (가설 부합)
- v3 (6) → stability **33.3%** (**가설 반박**)

**해석**: Grid 축소가 자동으로 stability 를 올리지 않는다.
- 6 조합 중 어느 것도 6/18 이상 선택되지 못함 → window 마다 실제로 다른 param 이 최적
- v2 의 12 조합 에선 한 조합이 9/18 지배 → 그 조합이 운 좋게 여러 윈도우에서 최적이었음
- **CSMOM 의 winning params 는 시간에 따라 진짜로 변한다** (regime/trend shift 에 반응)
- 고정 grid 내에서 "stability 올림" 은 grid 설계가 아닌 **strategy 본질적 한계**

→ 더 좁은 grid (예: 3 조합) 로도 stability 60% 는 기대하기 힘듦. 다른 축에서 해결해야 함
(adaptive selection / multi-lookback ensemble / 전략 구조 재설계).

## selection alpha 유지 여부

| 모델 | strategic_avg | incremental vs baseline |
|---|---:|---:|
| v1 | +1.01% | **+7.80%** |
| v2 | -1.28% | +5.51% |
| v3 | **-1.68%** | +5.10% |
| baseline | -6.79% | +0.00% |

**Selection alpha 는 3 버전 모두에서 baseline 대비 +5%p 이상 유지** — CSMOM 이
"단순 regime overlay" 가 아님은 재확인.

하지만:
- strategic_avg 가 v1 (+1%) → v2 (-1.3%) → v3 (-1.7%) 로 연속 악화
- grid 축소가 edge 를 희석시키는 현상 확인
- v1 의 +1% 는 lookback=30 덕이고, 그걸 제거하면 edge 도 같이 사라짐
- = "CSMOM 의 strategic edge 는 공격적 lookback 에서만 나오지만, 그건 winner-chasing 과 구분 어려움"

장기 과제이므로 RCDB 로 edge 축 전환하는 것이 합리적.

## 최종 판정

### **STOP — CSMOM (현 고정-grid 구조) 중단, RCDB 로 이동**

**STOP 근거 (not REVISE/HOLD)**:

1. **PASS 이중 hard-fail**: stability (33.3%) + strategic_avg (-1.68%) 둘 다 fail.
2. **Grid 축소 방향 실패 확인**: v1(27.8) → v2(50) → v3(33.3) 비단조 → 같은 축의 v4 의미 없음.
3. **strategic edge 가 grid 좁힐수록 감소**: +1% → -1.3% → -1.7% 일관적 악화.
4. **문제 본질**: winning params 가 시간에 따라 이동 → 고정 파라미터로 실전 운용 불가
   → 해결은 "grid 조정" 이 아닌 "전략 구조 자체 재설계" (adaptive / multi-lookback ensemble).
5. **Selection alpha 는 존재하지만 "실전 파라미터 선택 불가" 상태로는 paper live 가치 없음**.

**not STOP 이라고 변호할 만한 점**:
- Incremental alpha +5.10%p 는 ATR(~0) 보다 명백히 높음
- CSMOM 자체의 가설은 기각되지 않음
- 단지 v1~v3 로 대표되는 "고정 grid + 단일 lookback" 구조가 실패한 것

→ **현 구조는 STOP, CSMOM 철학(rank+regime) 자체는 향후 재설계 대상으로 보존**.
체크리스트 B3 를 STOP 으로 마감하되, 별도 "CSMOM v-next 재설계 장기 과제" 로 분리.
