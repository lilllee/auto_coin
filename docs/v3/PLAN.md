# auto_coin V3 — 실운영 의사결정 지원 및 전략 검토 고도화 계획서

작성일: 2026-04-15 · 승인 후 `main` 브랜치에서 구현/누적 검증 진행.

---

## 0. 배경

V2까지의 목표는 **모바일 웹 운영 콘솔 구축**이었다.  
그 결과, V1의 CLI/SSH 중심 운영은 웹 UI로 거의 완전히 대체되었고, 설정 변경·대시보드 확인·차트 조회·로그 확인·리포트 열람이 모두 모바일에서 가능해졌다.

또한 V2 후반 작업으로 전략별 검토를 위한 `review` 기능이 추가되었고, 다음과 같은 점이 확인되었다.

- 현재 `review`는 **전략 신호 replay** 중심이다.
- 실운영은 API를 통해 **현재가(current_price)** 를 기준으로 판단하지만,
  `review`는 주로 **일봉/종가 기반 replay** 이다.
- 전략별 구현을 확인한 결과,
  다수 전략은 설계상 **진입 신호 중심(entry-heavy)** 이며,
  청산은 RiskManager / 시간청산 / 운영 계층이 담당한다.
- 따라서 현재 `review`만으로는
  **“실제로 언제 사고 언제 파는지”**
  를 투자 관점에서 충분히 설명하지 못한다.

즉, V2가 **운영 콘솔**과 **전략 검토 기반**을 만든 단계였다면,  
V3는 그 위에서 한 단계 더 나아가

> **“실운영과 투자 판단에 실제로 도움 되는 시스템”**

을 만드는 단계다.

---

## 1. V3 목표

V3의 핵심 목표는 다음 3가지다.

1. **실운영과 전략 검토(review)의 간극 축소**
2. **현재 전략/시장 상태를 사람이 이해하기 쉽게 설명**
3. **실제 투자 판단에 도움이 되는 정보 제공**

한 줄로 정리하면:

> V2가 “운영 가능한 콘솔”이었다면, V3는 “운영 + 판단 보조 시스템”이다.

## 3. V3 핵심 문제 정의

### 3-1. 현재 review의 한계

현재 `review` 기능은 전략 구현과 정합하다는 장점이 있지만, 다음 한계가 있다.

- 많은 전략이 기본적으로 **SELL을 직접 내지 않음**
- 일부 전략은 `allow_sell_signal=false` 가 기본값
- 실운영의 손절 / 시간청산 / RiskManager와 분리되어 있음
- 일봉 replay는 장중 신호를 충분히 반영하지 못할 수 있음

결과적으로 사용자는 review 화면을 보고 다음과 같이 느끼기 쉽다.

- “왜 BUY 한 번 후 HOLD만 반복하지?”
- “이 전략은 실전에서도 몇 달 동안 매수 안 하나?”
- “실시간 운영과 review가 왜 이렇게 다르지?”

이 문제는 V3에서 반드시 다뤄야 한다.

---

### 3-2. 실운영에서 진짜 중요한 질문

실전 투자/운영 관점에서 중요한 것은 단순히 “전략이 있느냐”가 아니다.

실제로 중요한 것은 아래 질문들이다.

- 지금 **왜 매수하지 않는가?**
- 지금 **어떤 전략이 활성 상태인가?**
- 지금 **리스크 오프 상태인가?**
- 지금 **운영상 청산이 걸렸는가, 전략상 청산이 걸렸는가?**
- review 결과와 실전 결과가 **왜 다른가?**

즉, V3는 전략 수를 늘리는 것보다  
**현재 전략의 상태를 더 잘 설명하고 비교하는 것**이 중요하다.

## 5. V3 우선순위

V3는 아래 순서로 진행하는 것이 가장 실용적이다.

### 1순위 — Review 실용성 개선
현재 전략 검토 화면을 “실제 의사결정에 더 가까운 화면”으로 개선한다.

### 2순위 — 실시간 전략 상태판
지금 당장 어떤 전략이 어떤 상태인지, 왜 안 사고 있는지 보여준다.

### 3순위 — 전략 비교/설명 기능
전략별 상태 차이와 성격 차이를 사용자가 이해할 수 있게 만든다.

### 4순위 — 운영 리스크 설명/통합
전략 청산과 운영 청산을 구분하고 시각화한다.

## 7. V3에서 하지 않을 것 (비범위)

V3는 아래 항목을 기본 범위에서 제외한다.

- [ ] 완전한 포트폴리오 백테스트 엔진 재작성
- [ ] 선물/레버리지/공매도 지원
- [ ] 대규모 프론트엔드 프레임워크 전환 (React/Vue)
- [ ] 실거래 체결 엔진 전면 재설계
- [ ] 데이터 웨어하우스 수준의 장기 통계 시스템
- [ ] 머신러닝/AI 기반 자동 전략 생성

즉 V3의 목적은 **전략 연구 플랫폼 확장**이 아니라,  
**현재 전략과 운영을 더 잘 이해하고 투자 판단에 연결하는 것**이다.

## 13. 승인 / 진행 요약

V3는 아래 원칙으로 진행한다.

- [ ] 기존 V2 운영 안정성을 깨지 않는다
- [ ] review의 설명력과 실전 정합성을 우선 개선한다
- [ ] 전략 수를 늘리기보다, 현재 전략을 더 잘 이해하게 만든다
- [ ] 실운영에 도움이 되는 상태판/설명 UI를 우선한다
- [ ] 실거래 엔진보다 “왜 사고 안 사고 파는지”를 먼저 설명한다

### 진행 순서 제안

1. V3.0 — 기준선 정리
2. V3.1 — review SELL 모드
3. V3.2 — reason 강화
4. V3.4 — Signal Board
5. V3.3 / V3.5 — 실운영 정합성 강화
6. V3.6 이후 비교/리스크 확장
