## 착시 효과 / 구조적 약점

### A. 착시 (in-sample → WF 붕괴 크기)

| 비교 | in-sample | WF (strategic) | 붕괴비율 |
|---|---:|---:|---|
| CSMOM v1 default | +88.09% | **+1.01%** | 88x 감소 |
| CSMOM v1 regime OFF | +53.65% | +2.10% | 26x 감소 |
| ATR 채널 (참고) | +58~270% | -0.90~+0.37% | ∞ (알파 0) |

→ **CSMOM 도 ATR 과 매우 유사한 붕괴 패턴**. 단지 strategic +1% 는 "0 아님" 정도 수준.

### B. 구조적 약점

1. **Best-param stability 27.8%** (hard-fail). 매 30일마다 winning params 가 바뀜 → 실전에서 "이번 기간에 어떤 lb/k 로 돌릴지" 근거 없음.
2. **regime_ma 민감도 과도** — 100 이 sweet spot, 50/150/200 에서 급격히 떨어짐 (+88 → +56, +48, +24).
3. **top_k=1 이 극단 상위** — single-asset 집중은 sampling noise 증폭. 실전 위험.
4. **최근 6 윈도우 flat 고정** — 2025-09 이후 regime-off 지속으로 trade 없음. WF 가 포트폴리오 동작을 평가할 material 을 못 만듦.
5. **MDD < -30% 조합이 72.5%** — default 주변만 MDD 가 괜찮음. 일부 파라미터에서 -42% 까지 튐.
6. **Universe 의존** — 5 종목 교집합 2y. 상장 시점 / survivorship 보정 없음.

### C. 구별해야 할 것

- "regime filter 의 flat-in-bear 효과" = legitimate alpha, 단 momentum 과 무관
- "cross-sectional momentum edge" = 실제 strategic excess +1.01% — ATR 의 0 보다는 나으나 pass target +2% 미달

## Stage 4 기준 충족 여부 최종

| 기준 | Hard fail | Soft warn | Pass target | 실제 | 판정 |
|---|---|---|---|---|---|
| avg_test_excess | ≤ 0 | 0 ~ +1% | ≥ +2% | +4.03% (strategic +1.01%) | **pass** (all) / **soft-warn** (strategic) |
| positive_excess_ratio | < 40% | 40~55% | ≥ 55% | 72.2% | ✅ pass |
| train_test_ratio | > 5x | 3x ~ 5x | ≤ 3x | 4.97x | ⚠️ soft-warn |
| test_trades aggregate | < 30 | 30~50 | ≥ 50 | 109 | ✅ pass |
| best_param_stability | < 30% | 30~60% | ≥ 60% | 27.8% | ❌ **HARD-FAIL** |

**보수적 해석 (사용자 요청 기준)**:
- hard-fail 1건 존재 (stability)
- soft-warn 1건 (T/T)
- "pass" 항목 중 avg_excess 는 flat 덕이 절반 이상

**즉 숫자로 pass 3 / soft-warn 1 / hard-fail 1, 단 pass 2개(avg_excess·positive_ratio)는 flat-in-bear 효과로 상당 부분 설명 가능**.

## 최종 판정

### **REVISE — CSMOM v1 은 현 설계로 paper live 이전 단계로 못 감**

### 답변

1. **CSMOM 은 현재까지 나온 전략 중 첫 실전 후보인가?**
   → **아니다** (아직은). ATR 처럼 명백한 0 알파는 아니지만 (strategic +1.01% 존재), best_param_stability hard-fail 로 "이번 달엔 뭘 돌릴지" 판단 근거 없음. 실전 투입 불가.

2. **또 하나의 in-sample 환상인가?**
   → **반만**. in-sample +88% → strategic WF +1.01% 는 88x 붕괴로 ATR 에 근접한 수준의 환상 요소. 다만 "regime filter flat-in-bear" 라는 **별개의 실제 alpha** 가 존재함이 확인됨. 이건 CSMOM 고유의 momentum edge 가 아니라 regime 덕.

3. **paper live 설계로 넘어갈 가치가 있는가?**
   → **지금은 No**. Param stability 확보 전에 paper 는 낭비. 먼저 REVISE 방향 탐색 필요.

4. **RCDB 를 병렬로 바로 시작?**
   → **Yes, 병렬 착수 권장**. CSMOM 의 REVISE 가 시간 걸리고, RCDB 는 다른 edge 축(regime + mean-reversion)이라 독립. 둘 중 하나라도 통과하면 다음 단계.

## REVISE 방향 제안 (다음 턴 시작점)

1. **Param grid 축소 · stability 개선**
   - `lookback` 고정 (60일 단독) · `top_k` 고정 (2 또는 3)
   - 줄어든 grid (예: regime_ma × rebal 만 탐색) 로 WF 에서 stability ≥ 60% 달성 가능한지 재확인

2. **Regime-only 전략 비교군** 추가
   - CSMOM 없이 "BTC SMA100 > → 100% equal-weight universe, 아니면 cash" 만 한 베이스라인
   - 이 베이스라인이 CSMOM 과 비슷한 결과면 momentum edge 는 없다는 뜻

3. **Strategic excess 지표 추가**
   - portfolio_walk_forward 결과에 "trades>0 windows only" 집계 추가
   - "flat-in-bear 덕" vs "momentum edge" 분리 표시

4. **Longer history**
   - 2.5 ~ 3 년 데이터로 WF 윈도우 수 증가 (현재 18 → 30+)

5. **대안: RCDB 병렬 착수**
   - CSMOM 의 param stability 문제 해결 전까지, RCDB (single-asset regime + dip-buy) 설계 문서 `docs/v4/PLAN_RCDB.md` 작성
   - BTC/ETH 한정 → universe 문제 회피, trade count 확보 용이
