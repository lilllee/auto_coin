# 전략 백테스트 개발 가이드

## 개요

이 가이드는 `auto_coin` 백테스트 러너를 멀티 전략 시스템으로 확장하는 방법을 설명합니다.

**현재 상황:**
- 백테스트 러너(`src/auto_coin/backtest/runner.py`)는 VolatilityBreakout(변동성 돌파)만 지원
- 전략 레지스트리(`strategy/__init__.py`)에 5가지 전략이 등록되어 있음
- 각 전략은 순수 함수 인터페이스(`Strategy.generate_signal()`) 구현

**목표:**
- 백테스트 러너를 전략에 구애받지 않도록 일반화
- CLI 인터페이스 확장으로 모든 전략 지원
- 웹 UI 백테스트 기능 준비 (향후)

---

## 1. 현재 백테스트 러너 구조

### 1-1. 핵심 로직 흐름

```
CLI 입력 → 캔들 조회 → 데이터 전처리 → 백테스트 시뮬레이션 → 결과 출력
  ↓         pyupbit      enrich_daily      backtest()      BacktestResult
```

### 1-2. 주요 함수

#### `backtest(df, strategy, *, fee, slippage) -> BacktestResult`

**역할:** 일봉 데이터로 전략을 시뮬레이션하고 매매 결과를 반환

**입력:**
- `df`: `enrich_daily()`로 전처리된 DataFrame (필수)
  - 컬럼: `open`, `high`, `low`, `close`, `volume`, `target`, `ma{window}`
- `strategy`: `VolatilityBreakout` 인스턴스
- `fee`: 매수/매도 수수료 (기본 0.05%)
- `slippage`: 슬리피지 (호가 미끄러짐)

**진입/청산 로직:**
1. 매 캔들(오늘)에 대해:
   - `high >= target` 확인 (돌파 여부)
   - MA 필터: `target > ma_window` (필터 활성화 시)
   - 조건 충족 → 진입가 = `target * (1 + slippage)`
2. 다음 캔들 시가에 청산:
   - 청산가 = `next_open * (1 - slippage)`
   - 수익률 = `(exit * (1-fee)) / (entry * (1+fee)) - 1`

**현재 문제점:**
- `target` 컬럼 하드코딩 (VB 전용)
- 진입 로직이 고정됨 (돌파 + MA 필터)
- 다른 전략의 신호 처리 불가

**코드 구조:**
```python
def backtest(df, strategy, *, fee=..., slippage=...):
    # 유효성 검사

        raise ValueError("df must be enriched")

    for i in range(len(df) - 1):
        row = df.iloc[i]
        target = row["target"]
        if high < target:
            continue  # ← VB 하드코딩
        # ... 진입/청산 로직 ...
        trades.append(Trade(...))

    return _build_result(trades)
```

#### `_run_one(df, k, ma_window, ...) -> BacktestResult`

**역할:** 단일 파라미터 세트로 백테스트 실행

```python
def _run_one(df, k, ma_window, fee, slippage, require_ma):
```

- `enrich_daily()` 직접 호출 (VB 전용)

### 1-3. 데이터 전처리 (enrich_daily)

```python
def enrich_daily(df, ma_window=5, k=0.5):
    """VB 전용 전처리.

    추가 컬럼:
        - range: 전일 (high - low)
        - target: 오늘 시가 + 전일 range * k
        - ma{window}: 종가 N일 이평
    """
    out = df.copy()
    out["range"] = (out["high"] - out["low"]).shift(1)
    out["target"] = out["open"] + out["range"] * k
    out[f"ma{ma_window}"] = out["close"].rolling(window=ma_window).mean().shift(1)
    return out
```

**VB 전용 설계:**
- `target`, `range` 컬럼은 VB 진입가 계산용
- 다른 전략은 이 컬럼이 필요 없음 (또는 다른 용도)

### 2-4. 진입/청산 로직 일반화

**현재 VB 진입 로직:**

```python
# backtest() 내부
row = df.iloc[i]
target = row["target"]
if high < target:
    continue  # 돌파 실패
# ... 진입 ...
```

**문제점:**
- `target` 컬럼 하드코딩
- 다른 전략은 다른 신호 판단 기준 필요
  - ATR 채널: `high > upper_channel`
  - SMA200: `close > sma200`
  - Donchian: `high > donchian_high_20`

**변경 방향:**

시뮬레이션 상황에서도 `strategy.generate_signal()`을 호출:

```python
def backtest(
    df: pd.DataFrame,
    strategy: Strategy,
    *,
    fee: float = UPBIT_DEFAULT_FEE,
    slippage: float = 0.0,
) -> BacktestResult:
    """전략에 구애받지 않는 백테스트."""
    trades = []
    has_position = False

    for i in range(len(df) - 1):
        row = df.iloc[i]
        current_price = float(row["close"])

        # 전략에 신호 요청 (상황에 맞게)
        snap = MarketSnapshot(
            df=df.iloc[:i+1],  # 지금까지의 데이터
            current_price=current_price,
            has_position=has_position,
        )
        signal = strategy.generate_signal(snap)

        # BUY 신호 처리
        if signal == Signal.BUY and not has_position:
            entry_price = current_price * (1.0 + slippage)
            # ... 진입 기록 ...
            has_position = True
            entry_i = i

        # SELL 신호 처리 (allow_sell_signal 있는 전략)
        elif signal == Signal.SELL and has_position:
            exit_price = current_price * (1.0 - slippage)
            # ... 청산 처리 ...
            has_position = False

        # 정기 청산 (다음 봉 시가에 무조건 청산 - VB 방식)
        elif has_position and i == entry_i + 1:
            exit_price = float(df.iloc[i]["open"]) * (1.0 - slippage)
            # ... 청산 처리 ...
            has_position = False

    return _build_result(trades)
```

**주의사항:**

1. **청산 방식의 이중성:**
   - VB: 다음 봉 시가 강제 청산 (시간 기반)
   - 다른 전략(allow_sell_signal=True): 신호 기반 청산

   해결 방법:
   ```python
   # 전략 메타데이터
   STRATEGY_HOLD_PERIOD = {
       "volatility_breakout": 1,  # 1봉 보유 후 자동 청산
       "sma200_regime": None,     # 신호 기반 청산만
       # ...
   }

   # backtest() 내부
   if STRATEGY_HOLD_PERIOD[strategy.name] is not None:
       hold_days = STRATEGY_HOLD_PERIOD[strategy.name]
       if i >= entry_i + hold_days:
           # 자동 청산
   ```

2. **SELL 신호 처리:**
   - `allow_sell_signal=True` 전략만 SELL 신호 생성
   - 백테스트에서도 이를 존중해야 함

   ```python
   # backtest() 내부
   elif signal == Signal.SELL and has_position:
       if getattr(strategy, "allow_sell_signal", False):
           # SELL 신호 처리
       else:
           # 무시 (이 전략은 SELL 신호 미지원)
           pass
   ```

```python
def backtest(
    df: pd.DataFrame,
    strategy_name: str,
    strategy_params: dict,
    *,
    fee: float = UPBIT_DEFAULT_FEE,
    slippage: float = 0.0,
) -> BacktestResult:
    """멀티 전략 백테스트.

    Args:
        df: 원본 OHLCV DataFrame (전처리 전)
        strategy_name: 전략 이름 (e.g., "volatility_breakout")
        strategy_params: 전략 파라미터 dict
        fee, slippage: 거래 비용
    """
    if df.empty:
        return BacktestResult()

    # 1. 전략 인스턴스 생성
    strategy = create_strategy(strategy_name, strategy_params)

    # 2. 데이터 전처리
    enriched = enrich_for_strategy(
        df,
        strategy_name,
        strategy_params,
        ma_window=5,  # 기본값
        k=0.5,        # VB용 기본값
    )

    # 3. 최소 캔들 검증
    config = STRATEGY_CONFIG.get(strategy_name, {})
    min_candles = config.get("min_candles", 60)
    if len(enriched) < min_candles:
        raise ValueError(
            f"{strategy_name} requires {min_candles} candles, "
            f"got {len(enriched)}"
        )

    # 4. 시뮬레이션
    trades = []
    has_position = False
    entry_i = -1
    entry_price = 0.0

    hold_period = config.get("hold_period_days", None)

    for i in range(len(enriched) - 1):
        row = enriched.iloc[i]
        current_price = float(row["close"])

        # 신호 생성
        snap = MarketSnapshot(
            df=enriched.iloc[:i+1],
            current_price=current_price,
            has_position=has_position,
        )
        signal = strategy.generate_signal(snap)

        # 진입 처리
        if signal == Signal.BUY and not has_position:
            entry_price = current_price * (1.0 + slippage)
            entry_i = i
            has_position = True

        # 신호 기반 청산
        elif signal == Signal.SELL and has_position:
            exit_price = current_price * (1.0 - slippage)
            next_row = enriched.iloc[i + 1]
            exit_date = enriched.index[i + 1]
            ret = (exit_price * (1.0 - fee)) / (entry_price * (1.0 + fee)) - 1.0
            trades.append(Trade(
                entry_date=enriched.index[i].to_pydatetime(),
                entry_price=entry_price,
                exit_date=exit_date.to_pydatetime(),
                exit_price=exit_price,
                ret=ret,
            ))
            has_position = False

        # 기간 기반 자동 청산 (VB 방식)
        elif has_position and hold_period is not None:
            if i >= entry_i + hold_period:
                # 다음 봉 시가에 청산
                exit_price = float(enriched.iloc[i + 1]["open"]) * (1.0 - slippage)
                exit_date = enriched.index[i + 1]
                ret = (exit_price * (1.0 - fee)) / (entry_price * (1.0 + fee)) - 1.0
                trades.append(Trade(
                    entry_date=enriched.index[entry_i].to_pydatetime(),
                    entry_price=entry_price,
                    exit_date=exit_date.to_pydatetime(),
                    exit_price=exit_price,
                    ret=ret,
                ))
                has_position = False

    return _build_result(trades)
```

```python
def cli(argv=None):
    # ... argparse ...

    raw = _fetch_candles(args.ticker, args.days)

    # 하위호환: VB 옵션이 지정되었으면 VB 모드 활성화
    if args.k is not None or args.ma_window is not None:
        strategy_name = "volatility_breakout"
        strategy_params = {
            "k": args.k or 0.5,
            "ma_window": args.ma_window or 5,
            "require_ma_filter": not args.no_ma_filter,
        }
    else:
        strategy_name = args.strategy
        try:
            strategy_params = json.loads(args.params)
        except json.JSONDecodeError as e:
            print(f"ERROR: Invalid JSON in --params: {e}", file=sys.stderr)
            return 1

    print(f"# {args.ticker}  strategy={strategy_name}  "
          f"candles={len(raw)}  fee={args.fee}")

    # 스윕 처리
    if args.sweep:
        param_name, start, stop, step = args.sweep
        values = _frange(start, stop, step)
        print(f"# Sweep {param_name}: {values}")
        print(f"{param_name:>10}  result")

        for value in values:
            params = strategy_params.copy()
            params[param_name] = value
            try:
                r = backtest(raw, strategy_name, params,
                           fee=args.fee, slippage=args.slippage)
                print(f"{value:>10.3f}  {r.summary()}")
            except Exception as e:
                print(f"{value:>10.3f}  ERROR: {e}", file=sys.stderr)
    else:
        # 단일 백테스트
        try:
            r = backtest(raw, strategy_name, strategy_params,
                       fee=args.fee, slippage=args.slippage)
            print(r.summary())
        except Exception as e:
            print(f"ERROR: {e}", file=sys.stderr)
            return 1

    return 0
```

## 4. 전략별 백테스트 주의사항

### 4-1. Volatility Breakout (변동성 돌파)

| 항목 | 값 |
|------|-----|
| 최소 캔들 | 60 (MA5 + range) |
| 추천 기간 | 180-365일 |
| 스윕 파라미터 | `k`: 0.3-0.7 (step 0.1) |
| 특성 | 1봉 보유, 다음 시가 청산 |

**전처리:** `enrich_daily()` (target, range, ma 계산)

**진입 조건:**
```python
current_price >= target  # target = open + range * k
current_price > ma_window  # MA 필터 (선택)
```

**청산:** 다음 봉 시가 무조건 청산

**백테스트 팁:**
- `k` 값이 작을수록 보수적 (진입 많음)
- `k=0.0`은 시가 진입과 동일
- MA 필터로 거짓 돌파 필터링

### 4-2. SMA200 Regime (추세 필터)

| 항목 | 값 |
|------|-----|
| 최소 캔들 | 250 (SMA200 + 충분한 여유) |
| 추천 기간 | 365-730일 |
| 스윕 파라미터 | `ma_window`: 100-300 (step 50) |
| 특성 | 신호 기반 청산 지원 |

**전처리:** `enrich_daily()` + `enrich_sma(ma_window)`

**진입 조건:**
```python
current_price >= sma * (1 + buffer_pct)
```

**청산:**
- 기본: 외부 스케줄러 (일일 청산)
- `allow_sell_signal=True`: 가격 < SMA일 때 SELL

**백테스트 팁:**
- 장기 추세 전략이므로 장기 데이터 필요
- `buffer_pct`로 오진 필터링
- SMA200은 보편적 장기 이평선

### 4-3. ATR Channel Breakout (변동성 채널)

| 항목 | 값 |
|------|-----|
| 최소 캔들 | 64 (ATR14 + 여유) |
| 추천 기간 | 180-365일 |
| 스윕 파라미터 | `channel_multiplier`: 0.5-2.0 (step 0.25) |
| 특성 | 신호 기반 청산 지원 |

**전처리:** `enrich_daily()` + `enrich_atr_channel(atr_window, channel_multiplier)`

**진입 조건:**
```python
current_price > upper_channel  # upper = low + atr * multiplier
```

**청산:**
- 기본: 외부 스케줄러
- `allow_sell_signal=True`: 가격 < lower_channel일 때 SELL

**백테스트 팁:**
- `channel_multiplier` 크면 큰 움직임만 포착
- `multiplier=1.0` (기본)은 ATR 폭에 기반한 채널
- volatility가 높은 종목에 유리

### 4-4. EMA+ADX Trend (추세추종)

| 항목 | 값 |
|------|-----|
| 최소 캔들 | 175 (EMA125 + ADX90) |
| 추천 기간 | 365-730일 |
| 스윕 파라미터 | `adx_threshold`: 10-25 (step 2.5) |
| 특성 | 신호 기반 청산 지원 |

**전처리:** `enrich_daily()` + `enrich_ema_adx(ema_fast, ema_slow, adx_window)`

**진입 조건:**
```python
ema_fast > ema_slow  # 상향 크로스
adx > threshold  # 추세 강도 확인
```

**청산:**
- 기본: 외부 스케줄러
- `allow_sell_signal=True`: EMA 데드크로스 (fast < slow)

**백테스트 팁:**
- ADX 임계값이 높을수록 강한 추세만 포착
- EMA 기간이 길어서 초기 데이터 많이 필요
- 추세 시장에서 성능 우수

### 4-5. AdTurtle (Turtle 기법)

| 항목 | 값 |
|------|-----|
| 최소 캔들 | 70 (Donchian entry 20) |
| 추천 기간 | 180-365일 |
| 스윕 파라미터 | `entry_window`: 10-40 (step 5) |
| 특성 | 신호 기반 청산 지원 |

**전처리:** `enrich_daily()` + `enrich_donchian(entry_window, exit_window)`

**진입 조건:**
```python
current_price > donchian_high_entry  # 20일 최고가 돌파
```

**청산:**
- 기본: 외부 스케줄러
- `allow_sell_signal=True`: 가격 < donchian_low_exit

**백테스트 팁:**
- `entry_window` > `exit_window` (진입이 청산보다 폭넓음)
- 전형적으로 entry=20, exit=10 권장
- 범위 거래 시장에서 여러 번 손절가능

## 9. FAQ

**Q: 왜 `backtest()` 시그니처를 바꿨나?**

A: 전략을 런타임에 선택할 수 있게 하기 위해. 이전에는 `Strategy` 인스턴스가 미리 생성되어야 했지만, 이제 이름과 파라미터로 동적 생성.

**Q: SELL 신호가 없는 전략은 어떻게 되나?**

A: `allow_sell_signal=False` 전략은 SELL 신호를 생성하지 않음. 청산은 외부 스케줄러(일일 리셋 등)에만 의존.

**Q: VB 방식(1봉 자동 청산)은 계속되나?**

A: 예. `STRATEGY_CONFIG["volatility_breakout"]["hold_period_days"] = 1`로 설정되어 있음.

**Q: 스윕 파라미터를 여러 개 하려면?**

A: 현재 한 번에 하나만 지원. 여러 파라미터 스윕은 향후 그리드 서치 기능 추가 필요.

**Q: 웹 UI는 언제?**

A: 이 가이드는 런너 구현에만 집중. 웹 UI는 별도 이슈(T16)에서 다룸.
